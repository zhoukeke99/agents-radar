## 3. Trend Signal Analysis

**Explosive Attention: Agent Memory and Skill Infrastructure**

The dominant pattern in today's data is the **shift from model-centric to agent-centric development**. While LLM repositories (Transformers, Ollama) maintain high absolute stars, the *velocity* is entirely in agent tooling—memory systems, skill frameworks, and stealth browsers for autonomous operation. This reflects a maturation: base models are "solved enough," and the community is racing to build the **operating system layer for persistent, capable agents**.

Three specific inflection points emerge. First, **"skills" as an abstraction** has overtaken "prompts" and even "agents" as the fundamental unit of exchange—exemplified by mattpocock/skills' 3,392-star day and the proliferation of `.claude` directory references. This mirrors software engineering's evolution from scripts to libraries to packages. Second, **memory persistence is now table stakes**, not experimental: agentmemory, claude-mem, and mem0 collectively demonstrate that stateless agents are insufficient for real tasks. The community is building **episodic memory with compression and relevance injection** as a core primitive.

Third, **stealth and sandbox infrastructure** (CloakBrowser, cua) signals agent deployment moving from demo to production—requiring bot evasion, cross-platform sandboxes, and security isolation. This connects to recent Claude Code and Codex releases, which assume but don't provide such infrastructure. The Rust language preference (openhuman, rig, supertonic) reflects performance demands of on-device, always-on personal AI. The absence of major new training frameworks and the dominance of application-layer tools suggests the open-source community has **pivoted from building models to deploying them autonomously**.
