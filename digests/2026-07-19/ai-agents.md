## Bug 与稳定性

今日报告的严重Bug (P0/P1) 集中在启动失败、数据竞争和运行时崩溃。

- **P0 (极高)**:
    - **[Bug]: update to openclaw 2026.7.1: gateway fails to start w/ error (#108435)** [Issue链接](https://github.com/openclaw/openclaw/issues/108435)
    - **描述**: 升级到2026.7.1后，Gateway在systemd、ollama、命令行等多种启动方式下均失败。**（已有PR #108533 提出修复）**
- **P1 (高)**:
    - **Memory dreaming run pegs the gateway event loop (#99910)** [Issue链接](https://github.com/openclaw/openclaw/issues/99910)
        - **描述**: `Memory Dreaming`功能会导致Gateway主事件循环被锁死约10分钟，导致服务不可用，短期记忆存储永远无法持久化。
    - **Context overflow 算错 (#108238)** [Issue链接](https://github.com/openclaw/openclaw/issues/108238)
        - **描述**: 2026.7.1版本中，上下文用量错误地将`cacheRead`计入`totalTokens`，导致误触发上下文超限和压缩失败。**（已有PR修复）**
    - **Codex PreToolUse hook CPU-bound (#91009)** [Issue链接](https://github.com/openclaw/openclaw/issues/91009)
        - **描述**: Codex集成中的`pre_tool_use` hook 会衍生大量高CPU消耗的进程，导致Gateway RPC停滞。
    - **Gateway crashes with ERR_INVALID_STATE (#99263)** [Issue链接](https://github.com/openclaw/openclaw/issues/99263)
        - **描述**: 在Node 26上，因Node.js的GC回收文件句柄而非显式关闭导致的Gateway进程崩溃。

- **优势**:
